Module quality checkpoint

先验收树和火山。

这版只做两个 Three.js 模块资产：目标树和目标火山。 我刻意不做完整地球，也不继续推进主场景，避免方向错了还扩大修改。

重点不是“加面数”。重点是：自定义 mesh、低面数切面、材质色块、体块连接、和地表接触。 目标是更接近你说的地表山脉品质，而不是几个棱锥拼起来。

你只需要判断：这棵树和这个火山，是否已经接近 Image 1 那种高质量 low-poly props 的标准。
custom three.js mesh
Target tree弯曲树干、根部、分层树冠、分面暗面/亮面。不是球插棍。
Target volcano宽底座、破碎火山口、暗内壁、局部熔岩、块状烟。不是锥体贴图。

Quality rules

这个模块页的验收标准

01 custom mesh

不靠 primitive 直接堆

树干、树冠、火山体、火山口都用自定义低多边形形体或经过雕刻的块体，不是单个 cone/sphere。

02 ground contact

资产要长在地表上

树有根部和投影，火山有火山岩地表扩散。模块不是插在地上的小摆件。

03 painted facets

材质由分面表达

暗面、亮面、边缘高光、局部暖色，不靠写实贴图，也不是纯单色塑料。

If approved

确认后迁移策略

Tree system

树会变成资产生成器

  • 用同一套树干/树冠 grammar 生成不同树种。
  • 经纬度/气候只决定位置、密度、尺寸和颜色。
  • 先在南美森林样板里验证，不全局乱铺。
Volcano system

火山会变成地貌模块

  • 真实火山经纬度决定位置。
  • 火山底座会和地表高度/颜色融合，不像单独插上去。
  • 只保留少量可读火山，不为了数量牺牲质量。
